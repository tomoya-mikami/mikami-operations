# 業務自動化ツール開発 整理メモ

最終更新: 2026-05-23

## 目的

Triad.inc の eBay 出品・リサーチ・送料計算・Sellsta CSV 管理を自動化し、少ない手作業で出品数を増やす。

## 全体像

現在の開発は大きく3系統に分かれている。

1. マスターファイル / 既存出品ポリシー変更ツール
   - 既存出品のShipping Policyを見直すためのGoogle Sheets + GASツール。
   - このチャットで多くの設計・実装・検証が進んだ。

2. リサーチサポートツール
   - 仕入候補からSellsta出品CSVを作るためのGoogle Sheets + GASツール。
   - 別チャットで開発中。
   - 送料計算・ポリシー判定部分をマスターファイル側の詳細データと統合したい。

3. マスター送料体系拡張
   - FedEx FICP以外に、国際eパケットライト、DHL、Pegasus、OC Economy、SpeedPAK Economyなどを比較対象にするための大改修。
   - 作業規模が大きいため、専用チャットで進める方針。

## 重要な判断

- カーパーツはSKUに「三神」を含むものとして扱う。
- カーパーツは重量・サイズの精度が重要で、現状のM1判定精度が低いため一旦保留。
- バリエーション出品はShipping Policyが空白になるため、今回のポリシー変更対象から除外。
- Sellsta CSV内にあるがeBay上ですでに出品終了済みの商品は、今回の編集対象から除外する。
- Sellsta上の終了済みデータ整理用に、Action=EndのCSVを作成する方針。

## Sellsta関連の整理

Sellsta CSVは eBay File Exchange より今回の作業に適している。

理由:

- 現在のShipping Policy列が含まれている
- バックアップCSVを作れる
- tokyotuneの実出品件数に近い

注意:

- Sellstaの編集CSVでは action は `Revice`。
- End処理では action は `End`。
- 空白列は空白で上書きされる可能性があるため、元CSVの重要列は保持する。
- Policy名はeBay登録済み名称と完全一致が必要。

作成済み:

- Sellsta Endテスト10件CSV
- Sellsta End本番431件CSV
- Sellsta削除予定リスト
- Sellsta削除予定完全版

## リサーチサポートツールとの関係

リサーチサポートツールは別チャットで開発中。

既存のリサーチツールはすでに以下を持つ:

- 商品名/URLからAI判定
- 重量推定
- カテゴリID推定
- VeROリスク判定
- 利益計算
- Sellsta CSV出力
- 外注実績ダッシュボード

ただし送料計算が簡易的な固定値ベースのため、マスターファイル側の詳細送料データを使いたい。

方針:

- リサーチ開発はリサーチチャットで行う。
- このチャットでは、必要な引き継ぎ情報と作業分割を管理する。
- リサーチツール側には、マスターの送料・ポリシー・ゾーン定義を取り込ませる。

## マスター送料体系拡張

リサーチチャット側から、以下の要件が出ている。

- `shipping_carriers` に JP_EPL を追加
- `shipping_rates` に以下を追加
  - JP_EPL
  - DHL_SPEEDPAK
  - PEGASUS_DHL
  - OC_ECONOMY
  - SPEEDPAK_ECONOMY
- `policy_master` をキャリア対応版に再設計
- 各キャリアごとのゾーン定義を整理
- `GET_SHIPPING_USD(weight_kg, country, carrier_id)`
- `FIND_CHEAPEST_CARRIER(weight_kg, country)`

現状:

- FedEx FICPのみ詳細料金表あり
- DHL/Pegasus/OC/SpeedPAK/EPLは未整備
- 比較最安計算は現時点では完全にはできない

方針:

- 専用チャット「マスター送料体系拡張」で進める。
- このチャットでは進捗管理と引き継ぎ管理を行う。

## チャット運用ルール

この秘書チャットはフェイとして、司令塔・タスク管理・進捗集約を担当する。

作業が大きくなる場合:

- フェイが別チャット化を提案する。
- フェイがチャット名と初回プロンプトを作る。
- 分けた作業の進捗・成果物・保存場所・次アクションをこのチャットに戻す。
