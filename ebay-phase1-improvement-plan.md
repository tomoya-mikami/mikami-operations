# eBay業務自動化 Phase1 改善案

最終更新: 2026-05-25

## 結論

Phase1は機能自体を増やす前に、「どこを押せば何が動くか」「何が成功で何が失敗か」「今月税理士へ出せるか」を見える化するのが先。GASエディタで関数を探して実行する運用から、スプレッドシートメニューまたはWebダッシュボード中心へ移す。

## 改善方針

1. 稼働中GASは壊さない。
2. 削除ではなく、まず表示・分類・テスト計画を整える。
3. 本番操作は「確認画面」または「ドライラン」を挟む。
4. 月次運用はチェックリスト化する。
5. 経営防御ダッシュボードへ、売上・費用・在庫・提出書類を接続する。

## 使いやすくする案

| 改善 | 内容 | 優先度 |
|---|---|---|
| スプレッドシートメニュー整備 | 「日次処理」「月次処理」「チェック」「ログ確認」をメニュー化 | 高 |
| 実行ボタン | よく使う処理だけボタン化 | 高 |
| 実行ログシート | 実行日時、処理件数、成功/失敗、エラー内容を一覧化 | 高 |
| ドライラン機能 | 実際に保存・移動・通知する前に予定一覧を出す | 高 |
| 月次チェックリスト | 7日、11日、15日、月末の確認項目を並べる | 高 |
| 税理士提出確認画面 | 提出書類69項目の不足・完了を一覧化 | 高 |
| Webダッシュボード | ブラウザで状態確認と一部実行を行う | 中 |
| エラー通知 | 失敗時だけ通知し、通常成功はログに残す | 中 |
| 請求書管理画面 | 請求書の追加、分類、重複確認、削除候補表示 | 中 |

## Webダッシュボード案

### 最初に作る画面

| 画面 | 目的 |
|---|---|
| ホーム | 今日の成功/失敗、今月の提出状況を一目で見る |
| 税理士提出 | 69項目の提出状況、不足書類、確認待ち |
| 請求書/領収書 | 保存件数、未分類、重複候補 |
| eBay費用 | eBayインボイス、円換算、手数料 |
| 外注費 | 請求書有無、集計額、確認待ち |
| 在庫ログ | 在庫更新、未反映、滞留 |
| 実行ログ | いつ何を実行し、成功/失敗したか |

### ダッシュボードで実行してよい操作

- チェックだけの処理。
- 対象件数の確認。
- ドライラン。
- ログ表示。
- 月次チェックリストの更新。

### ダッシュボードで確認を挟む操作

- ファイル移動。
- ファイル名変更。
- 通知送信。
- 大量書き込み。
- チェックリスト復元。
- 削除候補の削除。

## 経営防御ダッシュボード接続案

| 指標 | データ源 | Phase1での接続 |
|---|---|---|
| 売上 | eBay売上一覧表 | 商品別・月別売上 |
| eBay手数料 | eBayインボイス | 請求額、手数料、広告費 |
| 円換算 | 円換算スプレッドシート | 月次円貨、為替 |
| 送料・関税 | eBay書類、送料マスター側 | Phase1単独では不足。送料マスターと接続 |
| 外注費 | 外注費一覧 | 月次外注コスト |
| 固定費 | メルカード利用明細、請求書 | カード明細・請求書から分類 |
| 領収書/請求書 | Drive保存データ | 経費根拠、提出漏れ |
| 在庫 | 在庫ログ | 滞留・未販売・仕入れ残 |
| 実質利益 | 売上、手数料、送料、関税、外注費、固定費 | ダッシュボードで統合計算 |

## 削除候補の扱い

削除候補4ファイルは、削除せずに次の観点で確認する。

1. トリガーに紐づいていないか。
2. 他ファイルから呼ばれていないか。
3. 現行12ファイルと役割が重複しているか。
4. 最終更新日と作成目的が分かるか。
5. 削除前にバックアップまたはエクスポートがあるか。

削除判断は、三神さん確認後に手動で行う。

## 実装順

1. Phase1監査メモを正本化する。
2. トリガー一覧と本番GASファイル一覧を確認する。
3. 実行ログシート案を作る。
4. スプレッドシートメニューを整理する。
5. 月次チェックリストを作る。
6. Webダッシュボード仕様書を作る。
7. Manus AIまたは別開発環境へ渡す。
