# 自律作業・権限確認メモ

最終更新: 2026-05-27

## 結論

三神さんは、データ削除、不可逆操作、セキュリティ/認証情報、個人情報/機密情報、Google Drive削除/移動、本番データ上書き、自動価格改定、自動出品停止を除き、AI側が確認なしで進めることを希望している。

Fay/Alfは、元データを壊さない別ファイル、別タブ、候補リスト、設計書、ドラフトコード、テストコード、作業ログ作成であれば止まらず進める。

## 確認不要

- 読み取り確認。
- 設計書、ロードマップ、チェックリスト、作業メモの作成。
- ドラフトコード、GAS案、テストコードの作成。
- 既存データを上書きしない新規ファイル作成。
- 既存列を壊さない新規シート/新規タブ追加。
- 候補リスト、見直し候補、アラート一覧、CSV案の生成。
- `handoff.md`、`projects.md`、`delegated-tasks.md` などの台帳更新。
- `git status`、commit、push。
- Alfへ渡す依頼文の作成。

## 確認必須

- 削除。
- 元データが失われる上書き。
- `reset`、`clean`、強制pushなどの不可逆操作。
- Google Drive内の削除、移動、リネーム。
- パスワード、口座番号、マイナンバー、顧客個人情報、APIキー、認証JSON、Secrets、トークン、サービスアカウント情報の確認・保存・表示。
- 外部サービスや本番業務データへの直接反映。
- 自動価格改定。
- 自動出品停止。
- eBay、Sellsta、Google Sheets本番データへ既存値を上書きする処理。

## Codexアプリの権限ポップアップについて

チャット内の運用ルールとは別に、Codexアプリや実行環境がネットワークアクセス、外部コマンド、サンドボックス外操作について権限ポップアップを出す場合がある。

このポップアップはAI側の判断だけでは完全には無効化できない。三神さんが許可してよい内容であれば、表示された許可ダイアログで「今後も許可」または同等の永続許可を選ぶ。

ただし、広すぎる許可は避ける。削除系、任意スクリプト実行、認証情報表示につながるものは永続許可しない。

## 判断方針

迷った場合は、次の順で進める。

1. 既存データを壊さない読み取り・別ファイル作成で進める。
2. 本番へ直接書く必要がある場合は、新規タブや候補リストに限定する。
3. 削除、不可逆、セキュリティ、認証情報、個人情報、本番値上書きに当たる場合だけ止める。
