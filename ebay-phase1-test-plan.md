# eBay業務自動化 Phase1 テスト計画

最終更新: 2026-05-25

## 結論

Phase1は稼働中システムなので、いきなり本番データで動かさない。まず「見るだけ」「ドライラン」「サンプル実行」「本番実行」の4段階に分ける。

## テスト段階

| 段階 | 内容 | 実行可否 |
|---|---|---|
| 0. 棚卸し | GASファイル、トリガー、メニュー、保存先を確認する | 実行してよい |
| 1. 読み取りテスト | 対象件数取得、フォルダ存在確認、シート存在確認 | 実行してよい |
| 2. ドライラン | 保存・移動・通知せず、予定処理だけログに出す | ドライラン関数がある場合のみ実行 |
| 3. サンプル実行 | テスト用メール/ファイル/CSVで実行 | 三神さん確認後 |
| 4. 本番実行 | 本番フォルダ・本番シートに反映 | 三神さん確認後 |

## 共通確認

- GASプロジェクトに本番12ファイルがあるか。
- 不要候補4ファイルが本当に存在するか。
- 各トリガーが想定時刻で設定されているか。
- `deleteRow` を使う処理がないか。
- Gmail検索で日本語ラベルを除外条件に使っていないか。
- 通知系の送信先やルーム情報をGitHubへ保存していないか。
- エラー時にログや通知が残るか。

## トリガー確認方法

1. Apps Scriptの「トリガー」画面を開く。
2. 関数名、実行頻度、時刻、失敗通知設定を一覧化する。
3. 次の想定と一致するか確認する。

| 処理 | 想定トリガー |
|---|---|
| 請求書メール自動保存 | 毎日 8:00 |
| 領収書メール自動処理 | 毎日 9:00 |
| 書類自動振り分け | 毎日 23:00 |
| 税理士書類自動収集 | 毎月7日 9:00 |
| eBayインボイス管理 | 毎月7日 10:00 |
| 外注報酬自動集計 | 毎月7日 11:00 |
| 外注請求書チェック | 毎月11日 9:00 |
| 税理士提出書類チェック | 毎月15日 9:00 |

## 失敗時に見る場所

- Apps Scriptの実行ログ
- Apps Scriptのエラー通知
- 対象スプレッドシートのログシートまたは管理シート
- Google Driveの保存先フォルダ
- Gmailの処理済みラベル
- 通知系の送信履歴

## 実行してよいテスト

- メニュー表示確認。
- トリガー一覧確認。
- 対象件数の読み取り。
- 保存先フォルダ/シートの存在確認。
- 通知なしのチェック処理。
- サンプルファイルだけを使う抽出テスト。

## 実行前確認が必要なテスト

- Driveファイルの移動。
- Driveファイル名の変更。
- 本番メールへの処理済みラベル付与。
- 本番シートへの大量書き込み。
- 通知送信。
- チェックリスト復元。
- 不要候補ファイルの削除。

## 月次テスト順

1. トリガー一覧を確認する。
2. メニュー表示を確認する。
3. 請求書/領収書の対象件数だけ確認する。
4. 税理士提出書類チェックを通知なしで実行する。
5. eBay円貨換算をサンプルで確認する。
6. 外注費集計を通知なしで確認する。
7. 書類振り分け・リネームは予定一覧だけ確認する。
8. 問題なければ、三神さん確認後に本番実行する。
