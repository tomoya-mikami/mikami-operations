# 経営防御ダッシュボード v1 設計メモ

最終更新: 2026-05-26

## 結論

三神さんが毎月の実質利益、支出、資金繰り、削減余地を短時間で判断できるように、まずはGoogle Sheetsベースの「経営防御ダッシュボード v1」を作る。

新しくゼロから数字を入力するのではなく、eBay業務自動化 Phase1で既にGoogle Driveとスプレッドシートへ保存・集計しているデータを入力元にする。

## 目的

- 売上は立っているが、実際に利益が残っているか分からない状態を解消する。
- 送料、関税、eBay手数料、外注費、固定費、サブスク、仕入れなどを分解して見る。
- 経営判断に必要な数字を、三神さんが細かい分析をしなくても見える形にする。
- 融資相談、コスト削減、商品カテゴリの継続/撤退判断につなげる。

## v1で見るべき数字

| 区分 | 指標 | 目的 |
|---|---|---|
| 売上 | 月別売上 | 売上規模の把握 |
| 粗利 | 売上 - 仕入原価 | 商品そのものの収益性を見る |
| 実質利益 | 売上 - 仕入 - 手数料 - 送料 - 関税 - 外注費 - 固定費 | 本当に利益が残っているかを見る |
| eBay費用 | eBay手数料、広告費、International fee等 | 売上に対して手数料が重すぎないかを見る |
| 送料・関税 | 国際送料、関税、DDP/DDU関連費用 | 価格設定やShipping Policyの見直しに使う |
| 外注費 | 月別外注費、担当者別費用 | 外注費の増減と採算を見る |
| 固定費 | サブスク、カード明細、請求書 | 削減候補を見つける |
| 在庫 | 在庫額、滞留在庫、未販売仕入れ | 資金が寝ている場所を見る |
| 資金繰り | 今月末/翌月末の資金見込み | 融資・支払い判断に使う |

## Phase1から使えるデータ源

| データ源 | 使える内容 | v1での使い道 | 注意 |
|---|---|---|---|
| eBay売上一覧表 | 売上、販売件数、商品情報 | 月別売上、商品別売上 | 手数料や送料との接続が必要 |
| 円換算スプレッドシート | eBay売上の円換算、為替 | 円ベースの月次売上 | MURC TTM取得失敗時の扱いを確認 |
| eBayインボイス | eBay費用、手数料、請求額 | eBay費用率、実質利益 | PDF/CSV形式差分に注意 |
| メルカード利用明細 | カード支払い、固定費、仕入れ候補 | サブスク・固定費棚卸し | MoneyForward CSV形式を確認 |
| 外注費一覧 | 外注費、請求書有無 | 月別外注費 | 個人情報は保存しない |
| 01_請求書メール | 請求書PDF/添付 | 固定費・支払証憑 | 書類種別判定が必要 |
| 04_税理士提出用 | 領収書・月次書類 | 経費根拠、提出漏れ | 税務判断は税理士確認 |
| 在庫ログ | 仕入れ・在庫・滞留 | 在庫資金、粗利補助 | SKU/商品カテゴリ接続が必要 |

## 足りない可能性が高いデータ

| 不足データ | 理由 | 対応案 |
|---|---|---|
| 商品別仕入原価 | 実質利益計算の中心 | 在庫ログ、仕入れ記録、カード明細、手入力補正を接続 |
| 商品カテゴリ | 商品別/カテゴリ別利益を見るため | SKUルール、商品名判定、既存カテゴリ表を使う |
| 送料の実績 | 利益率に大きく影響 | 送料マスター v2.0、出荷実績、請求書を接続 |
| 関税の実績 | 関税問題の影響把握に必要 | 関税支払データ、eBay/配送請求、手入力補助 |
| 返品・返金 | 実質利益を下げる | eBay取引データから抽出できるか確認 |
| サブスク一覧 | コスト削減に直結 | カード明細/請求書からAI分類 |

## ダッシュボード画面案

### 1. 経営サマリー

見るもの:

- 今月売上
- 今月実質利益
- 利益率
- 固定費
- 資金繰り危険度
- 今月の要注意項目

判断:

- 今月は黒字か赤字か
- すぐ削る支出があるか
- 融資・支払い・仕入れを抑えるべきか

### 2. 利益分解

見るもの:

- 売上
- 仕入原価
- eBay手数料
- 送料
- 関税
- 外注費
- 固定費
- 実質利益

判断:

- 利益を削っている最大要因は何か
- 送料/関税/手数料/外注費のどこを改善すべきか

### 3. 商品カテゴリ別

見るもの:

- ゲーム機
- 無線機
- ブランド品
- カーパーツ
- カーオーディオ
- その他

判断:

- 売上は大きいが利益が薄いカテゴリ
- 小さくても利益率が高いカテゴリ
- 一時停止、価格改定、国内販売へ回す候補

### 4. 支出・サブスク

見るもの:

- 月額固定費
- サブスク
- 重複サービス
- 使っていない契約
- 前月比で増えた費用

判断:

- 今月削れる支出
- 解約候補
- 税理士/スタッフに確認する費用

### 5. 月次チェック

見るもの:

- eBayインボイス取込済みか
- 円換算済みか
- 外注費集計済みか
- 領収書/請求書保存済みか
- 税理士提出チェック済みか

判断:

- 月次締めのどこで止まっているか
- 三神さんが確認するべきものは何か

## 実装方針

### Phase A: 読み取り確認

- Phase1本番GAS 12ファイルの実在確認。
- トリガー確認。
- Script Propertiesはキー名だけ確認。
- ダッシュボード/メニュー有無を確認。
- 削除、変更、実行はしない。

成果物:

- `ebay-phase1-readonly-check-result.md`

状態:

- 済。本番GAS 12ファイル、不用候補4ファイル、トリガー、`runInvoiceBridge` の役割を確認済み。
- Script Propertiesは値露出リスクのため未確認。

### Phase B: データソース棚卸し

- 各スプレッドシートのシート名、列名、月次更新状況を確認。
- Driveフォルダはファイル削除・移動せず、構成と保存状況だけ確認。
- 経営防御に使える列と足りない列を分ける。

成果物:

- `management-dashboard-data-inventory.md`

状態:

- 一部済。Driveフォルダ構造、`00_要確認`、スクショOCRスクリプトは確認済み。
- スプレッドシート実列名、最新月データ、関税実績、商品別仕入原価は未確認。

### Phase C: v1シート設計

- Google Sheetsで最小ダッシュボードを作る前提で、シート構成を決める。
- 最初は自動連携より、既存シート参照 + 手入力補正を許容する。

候補シート:

- `経営サマリー`
- `月次利益`
- `商品カテゴリ別`
- `支出サブスク`
- `資金繰り`
- `データ取込状況`
- `要確認`

成果物:

- `management-defense-dashboard-v1-spec.md`

### Phase D: 自動化

- 既存Phase1を壊さず、参照用GASまたは別スプレッドシートで集計する。
- 既存GASに直接改修を入れる前に、読み取り専用の別集計を優先する。
