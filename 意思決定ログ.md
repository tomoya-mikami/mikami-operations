## 2026-05-23

### 安全作業の確認省略

決定:

- 引き継ぎファイル、台帳、ログ、プロジェクトメモの作成・更新は、毎回確認せずFayが進めてよい。
- Gitの `status`、commit、pushも、安全な保存作業としてFayが進めてよい。
- Alfへ渡す依頼文の作成も、毎回確認不要とする。
- 確認が必要なのは、削除、不可逆操作、個人情報/機密情報へのアクセスや保存、Google Drive内の削除・移動。

理由:

- 一つずつ確認すると作業効率が落ちるため。
- 引き継ぎの正確性を高めるには、進捗が出た時点で自動的に保存・同期する方がよいため。

根拠ファイル:

- `START_HERE.md`
- `handoff.md`
- `handoff-risk-register.md`

- リスクと対策を明文化しておくことで、次チャットやAlf側でも同じ判断ができるため。

根拠ファイル:

- `handoff-risk-register.md`
- `START_HERE.md`
- `handoff.md`

### 新チャット起動プロトコル

決定:

- 新しいチャットや別端末で再開するときの起点として `START_HERE.md` を作成する。
- 三神さんが新チャットで言う最小文は「Fay、START_HERE.mdから再開して。」でよい。
- 毎回GitHub同期を必須にしない。
- GitHub同期は、Alfが作業してpushした後、FayがAlfへ渡す前、端末をまたぐとき、重要な引き継ぎファイルを更新したときに行う。

理由:

- 毎回長い設定・引き継ぎ文を貼り直すと作業効率が落ちるため。
- 同じ端末・同じ作業台で続ける場合は、ローカルの `START_HERE.md` と `handoff.md` を読めば再開できるため。
- GitHubは共通作業台として重要だが、毎チャットごとに同期を要求すると運用負荷が高いため。

根拠ファイル:

- `START_HERE.md`
- `handoff.md`
- `AGENTS.md`

### 共通作業台 / Alf・Fay運用

決定:

- GitHubリポジトリ `mikami-operations` を、スマホ・ノートPC・デスクトップPC・Codex・Claude間の共通作業台として使う。
- まずは無料プラン・Privateリポジトリで運用する。
- Google Driveは資料保管、GitHubは作業ログと引き継ぎ台帳として使う。
- デスクトップPCは Alf / アルフ / Alfred / アルフレッド、ノートPCは Fay / フェイ / Friday と呼ぶ。
- 三神さんが「アルフでやって」「Alfで進めて」と言った場合は、デスクトップPC側Codexが `/Users/mikami/Documents/GitHub/mikami-operations-official` で作業する。
- Fayは司令塔として依頼を受け、長時間作業・重いファイル作業・調査・commit/pushが必要な作業はAlfへ分けることを提案する。
- Alfでは作業後に `handoff.md` 更新、commit、可能ならpushまで行う。
- Alf初回セットアップと毎回の作業依頼テンプレートは `desktop-codex-prompt.md` に保存する。

理由:

- 複数端末・複数AI間で作業内容が迷子になるのを防ぐため。
- 出張中やスマホ利用時にも、デスクトップPC側の作業を継続できるようにするため。

根拠ファイル:

- `handoff.md`
- `delegated-tasks.md`
- `desktop-codex-prompt.md`

### 引き継ぎ・オートセーブ運用

決定:

- 長期利用では、チャットが文脈上限で突然使えなくなる前に、余裕を持って引き継ぎメモを更新する。
- 進捗・決定・未完了タスクは、普段からローカル管理ファイルへ残す。
- 次スレッドの正本は `handoff.md` とする。
- 分割作業やアルフへの依頼は `delegated-tasks.md` に登録する。
- 重要な意思決定は `意思決定ログ.md` に残す。
- アルフ側の作業も、成果物・次アクション・未完了事項をフェイ側へ戻す運用にする。

理由:

- 前チャットが文脈上限で返信不能になり、会話本文を自動引き継ぎできないことが分かったため。
- 同じプロジェクト内でも、チャット本文そのものは完全には引き継がれないため。
- ローカルファイルに残した内容は、次スレッドで読み直して復元できるため。

根拠ファイル:

- `AGENTS.md`
- `delegated-tasks.md`
- `handoff.md`
