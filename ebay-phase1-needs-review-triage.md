# eBay業務自動化 Phase1 `00_要確認` 分類メモ

作成日: 2026-05-26

## 結論

`04_税理士提出用/00_要確認` は、自動振り分けが判断できなかったファイルの置き場。

見える範囲で50件あり、主に以下が混ざっている。

- FedEx請求書
- サブスク/サービス系の請求書・領収書
- 配送/通関/納品ラベル系
- 古いCSV/Excel
- 振込明細
- メール装飾画像
- ZIP

削除、移動、リネーム、ファイル本文閲覧、GAS実行はしていない。  
個別の人名、メールアドレス、請求書番号、ファイルIDはこのメモに保存していない。

## 間違いっぽいもの・迷子候補

| 優先 | 候補 | 理由 | 次の扱い |
|---|---|---|---|
| 高 | 2024年のFedEx請求書14件 | 月が読める請求書なのに `00_要確認` に残っている | 月別フォルダへ移動候補。ただし保存先カテゴリ名を決める |
| 高 | サブスク/サービス系の請求書・領収書14件 | 請求書/領収書として月が読める | `サブスク` / `各種請求書` へ移動候補 |
| 中 | 2022年の古い配送/CSV/Excel系 | 提出対象か、保管だけでよいか不明 | 税理士提出対象か確認 |
| 中 | Refund/CreditNote | 請求書とは逆方向の書類 | 返金/クレジットノートとして分ける候補 |
| 中 | メール装飾画像らしき画像 | 税務根拠にならない可能性が高い | 保存対象外候補。削除は三神さん確認後 |
| 低 | ZIP | 中身が不明 | 開かず保留。必要なら中身確認の許可を取る |

## おすすめの次アクション

1. `00_要確認` の中身は、いきなり動かさない。
2. まず「移動予定リスト」を作る。
3. 移動予定リストは、ファイル名をそのまま載せず、必要なら番号付き・マスク済みにする。
4. 三神さん確認後に、手動で移動するか、GASのドライランを作ってから移動する。

## スクリーンショット読み取りスクリプト

入っている。該当ファイルは `screenshot_data_extractor.gs`。

確認できた関数:

- `setupScreenshotFolders`
- `extractAllScreenshots`
- `extractJREBank`
- `extractMercariSales`
- `extractMercariPoints`
- `extractMercariBalance`
- `extractMercard`
- `extractAmazonPayments`
- `testOcrSingle`
- `testScreenshotExtractor`

役割:

- スクリーンショットやPDFをOCRで読み取る。
- メルカリ売上、メルカリポイント、メルカリ残高、メルカード、Amazon支払い、JREバンク明細などを想定している。
- 読み取った内容をスプレッドシートへ整理し、税理士提出用フォルダへつなぐ設計。

注意:

- 正本メモ上、Drive API v3 OCRを使う。
- JREバンクはOCR精度問題でスキップ状態と記録されている。
- スクショ/PDF読み取りは、実行するとファイル読み取りやシート更新が起きる可能性があるため、実行前に三神さん確認が必要。
