# 修理管理シート 改修実装パケット

最終更新: 2026-05-27

## 目的

修理管理シートを、外注さんが入力しやすく、社内側が報酬・原価・月次集計を確認しやすい形へ改善する。

## 成果物

実データを含む成果物本体はGitHubに保存しない。

- 確認用xlsx: `/private/tmp/修理管理シート_デザイン改善案_v2_2026-05-27.xlsx`
- 外注さん説明スライド: `/private/tmp/外注修理報酬ルール説明_v2_2026-05-27.pptx`
- Google Sheets反映用GAS: `repair-sheet-redesign-apply.gs`

## xlsx構成

| シート | 目的 |
|---|---|
| 使い方 | 入力ルールと運用の説明 |
| 修理記録 | 外注さん入力、自動計算、社内管理を色分けした実績入力 |
| 月次集計 | 返送月基準で報酬・件数・原価・要確認件数を集計 |
| 報酬ルール | 基準時給、上限、インセンティブ条件を管理 |

## 修理記録の列設計

### 外注さん入力

- 修理ID
- 作業日
- 返送日
- 商品カテゴリ
- 本体名称
- 色
- シリアル
- 実作業時間(分)
- 修理完了
- 修理完了日
- 修理後状態
- 修理内容
- 新品部品名
- 備考

### 自動計算

- 報酬計上月
- 作業報酬
- インセン対象
- 要確認

### 社内管理

- 部品代
- 仕入れ金額
- 送料
- 商品原価合計
- 総コスト

## 報酬ルール

- 動作確認とクリーニングは毎回実施前提のため、個別報酬項目にしない。
- 基準時給は1,500円/時。
- 動作確認済みとワケありは、1商品あたり上限1,500円。
- ジャンクは、1商品あたり上限1,000円。
- ワケありはインセンティブ対象外。
- インセンティブ対象は、修理完了かつ修理後状態が動作確認済みの商品。
- インセンティブは、対象30件ごとに1,000円。
- 返送日が入った月を報酬計上月とする。

## 入力UIの仕様

- `修理完了` はチェックボックス。
- Google Sheets上では、`修理完了` にチェックを入れた日に `修理完了日` が自動入力される。
- `修理完了` のチェックを外すと、`修理完了日` も空に戻る。
- `修理後状態` は `動作確認済み`、`ワケあり`、`ジャンク` のプルダウン。
- `部品代` は外注さんの入力欄ではなく、後半の `社内管理` 欄で管理する。

## GASの使い方

1. 修理管理のGoogle Sheetsを開く。
2. 拡張機能 > Apps Script を開く。
3. `repair-sheet-redesign-apply.gs` の内容を貼り付ける。
4. `createRepairManagementRedesign()` を実行する。
5. 作成された `_新` 付きシートを確認する。

GASは既存の `修理記録一覧` を変更しない。新しいタブとして `使い方_新`、`修理記録_新`、`月次集計_新`、`報酬ルール_新` を作成する。
