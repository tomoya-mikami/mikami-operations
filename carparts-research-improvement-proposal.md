# カーパーツリサーチ改善提案

最終更新: 2026-05-27

## 結論

三神さんが今困っている本質は、出品数を増やしたいのに、商品点数が増えるほど「仕入価格変動」「取扱終了」「送料ミス」「出品外注費」「価格見直し漏れ」が増え、売上を伸ばすほど赤字リスクも増えること。

解決方針は、作業を増やすことではなく、リサーチから出品までを次の5段階に分け、AIと自動チェックで「人間が判断する場所」を絞ること。

1. 型番収集。
2. 仕入可否・価格確認。
3. 利益・送料・リスク判定。
4. 出品データ生成。
5. 出品後の価格監視・販売停止候補抽出。

最初にやるべきことは、出品自動化より先に、赤字を防ぐ安全装置を作ること。価格監視MVP、Shipping Policy候補、要確認キュー、出品CSV生成をつなぐと、出品数を増やしても崩れにくくなる。

## 現在の問題点

| 問題 | 起きていること | 影響 |
|---|---|---|
| 仕入価格の変動 | モノタロウ価格が上がっても気づかない | 売れてから赤字 |
| 取扱終了・在庫切れ | 売れた後に仕入れできない | キャンセル、評価悪化 |
| 送料Policyミス | サイズ/重量不明のまま選ぶ | 送料赤字 |
| 推定送料と実送料の差 | 売れた後に送料超過が分かる | 利益が消える |
| 関税請求の見落とし | 請求額が想定より高くても原因が残らない | 利益率が下がる |
| Chatwork情報の反映漏れ | 品切れ、値上げ、廃番情報が出品リストに反映されない | 売れた後に仕入不可/赤字 |
| 基準セラー価格管理 | 基準セラー総額を下回る恐れ | 価格秩序が崩れる |
| 外注依存 | 出品データ作成を外注に頼る | 出品数が増えるほど費用増 |
| データ品質のばらつき | 型番、入数、価格、URLにミスが出る | 出品判断が止まる |
| 既存出品の見直し不足 | 古い価格/送料のまま残る | じわじわ利益が削られる |
| 商品選定の基準不足 | 売れにくい商品も混ざる | 出品件数は増えるが売上が伸びない |

## まず作るべき仕組み

| 優先 | 仕組み | 効果 |
|---|---|---|
| 1 | 価格監視MVP | 値上がり・取扱終了・在庫切れを検知 |
| 2 | 要確認キュー | 人間が見るべき商品だけに絞る |
| 3 | 利益率アラート | 赤字・薄利出品を止める |
| 4 | Shipping Policy候補 | 送料赤字を減らす |
| 5 | 送料差額チェック | 推定送料と実送料のズレを次回出品に戻す |
| 6 | 関税チェック | 想定外の関税請求を利益計算へ反映 |
| 7 | Chatwork情報取込 | 品切れ・値上げ・廃番情報を見落とさない |
| 8 | 出品CSV自動生成 | 外注費を減らし、出品数を増やす |
| 9 | 既存出品見直しレポート | 過去出品の赤字化を発見 |
| 10 | 売上・回転率ダッシュボード | 売れる型番/カテゴリへ集中 |

## AIを入れると効く場所

| 工程 | AIでできること | 人間が見るところ |
|---|---|---|
| 型番抽出 | eBayタイトル/説明から型番候補を抽出 | 型番が本当に一致するか |
| モノタロウ照合 | 型番、メーカー、商品名の一致度判定 | 完全一致しない商品 |
| 入数判定 | 商品説明から「2個入」「左右セット」を抽出 | 曖昧な表記 |
| 商品説明生成 | eBay用タイトル、説明文、注意書きを生成 | 禁止表現、適合表現 |
| 重量/サイズ推定 | 部品種別から梱包サイズ候補を推定 | 高額・大型・不明品 |
| Shipping Policy候補 | 重量帯/価格帯から候補Policyを出す | 例外品 |
| 価格改定候補 | 仕入価格上昇後の推奨販売価格を出す | 実際に改定するか |
| Chatwork情報抽出 | 品切れ、値上げ、廃番、納期未定と品番候補を抽出 | 自社SKUに反映するか |
| 関税候補 | HS/HTS候補、確認観点、根拠URL候補を出す | HTS分類と請求妥当性 |
| 出品優先度 | 売れ筋、利益率、競合数でスコア化 | 仕入戦略判断 |

AIに任せるべきではないこと:

- 型番が違う商品を同一と確定すること。
- 出品可否の法的/ポリシー判断を最終決定すること。
- 価格改定や出品停止を自動実行すること。
- 重量/サイズを確定値として扱うこと。
- 関税分類や請求妥当性を最終判断すること。

## 追加の赤字防止ループ

価格監視だけでは、送料と関税のズレを止めきれない。今後は次の4つを `出品反映候補` に集約する。

| 発生元 | 見ること | 候補化する条件 | 対応 |
|---|---|---|---|
| モノタロウ | 価格、在庫、取扱終了 | 値上げ、取扱終了、納期未定 | 価格見直し/出品停止候補 |
| 送料差額 | 推定送料、実送料 | 差額1,000円以上、または20%以上 | Shipping Policy見直し |
| 関税チェック | 想定関税額、請求額 | 差額1,000円以上、または20%以上 | 利益率/申告/HTS確認 |
| Chatwork | 品切れ、値上げ、廃番情報 | 品番候補とリスク語句がある | 仕入先確認/出品停止候補 |

- `gas/carparts_cost_control_mvp.gs`: 追加タブと候補集約を行うGAS MVP。

## 出品数を増やして安定売上にする方法

### 1. 出品候補を3ランクに分ける

| ランク | 条件 | 運用 |
|---|---|---|
| A | 売れ筋、利益率高、軽い、小さい、仕入安定 | 優先出品 |
| B | 利益はあるが競合多い/送料不安あり | 要確認後に出品 |
| C | 重い、薄利、取扱不安、適合不明 | 保留または国内販売候補 |

全件を同じ力で出すのではなく、Aランクを毎週増やす。

### 2. 売れる条件を数値化する

見る指標:

- eBay Product Researchの平均販売価格。
- 過去販売件数。
- Sell-through。
- 競合数。
- 送料込み総額。
- 想定利益率。
- 仕入先の安定性。
- 重量/サイズリスク。
- 返品/適合リスク。

### 3. 出品後の見直しを自動化する

出品したら終わりではなく、次を定期的に見る。

- 仕入価格が上がった商品。
- モノタロウ取扱終了の商品。
- 利益率が下がった商品。
- 60日以上アクセス/Watch/販売が弱い商品。
- Shipping Policyが古い商品。
- 基準セラー総額を下回りそうな商品。

## 一括出品の方法

公式に確認した選択肢:

| 方法 | 向いている状況 | コメント |
|---|---|---|
| Seller Hub Reports | CSV/Excelで追加・修正・終了をしたい | まずここが現実的 |
| eBay Bulk Listing Tool | 既存出品を画面上でまとめて編集 | 2,000件程度までの画面操作向け |
| Merchant Integration Platform | SKU単位で大量在庫を継続管理 | 5万SKU規模も視野。導入は中級以上 |
| Trading API / Sell Feed API | 開発して完全連携したい | 高度。今すぐより後工程 |
| Sellsta CSV | 現行運用に近い | 既存ツールとの接続がしやすい |

現時点の推奨:

1. まずSellsta CSVの受入テストを完了する。
2. 次にSeller Hub Reports形式にも出せるように列変換を用意する。
3. 出品データ生成はAI + GAS/ツールで行い、人間は「出品OK」だけ判断する。
4. 外注出品作業は、手入力作業から「CSV確認・例外処理」に縮小する。

## 外注費を減らす方法

| 現在 | 改善後 |
|---|---|
| 外注が1件ずつ出品データを作る | シートからCSVを自動生成 |
| 外注が説明文を考える | AIがタイトル/説明文案を生成 |
| 外注が送料Policyを選ぶ | 送料マスターが候補を出す |
| 外注が画像/URL/価格を確認 | 例外だけ手動確認 |
| 外注が完了報告する | 取り込みログで自動記録 |

外注さんの仕事を減らすというより、「人間でないと危ないところ」に寄せる。

残すべき人間作業:

- 型番が完全一致しない商品の判断。
- 適合・互換が不明な商品の確認。
- 高額・大型・送料不安商品の確認。
- 出品前の最終OK。
- クレームや返品につながりそうな表現チェック。

## リスクと回避方法

| リスク | 影響 | 回避方法 |
|---|---|---|
| 型番違い | 返品、クレーム | 型番完全一致、不一致は要確認 |
| 互換/適合ミス | 返品、低評価 | Fitment情報を別管理、AIは候補まで |
| 仕入価格上昇 | 赤字 | 価格監視、利益率アラート |
| 取扱終了 | キャンセル | 取扱状態監視、出品停止候補 |
| 送料ミス | 送料赤字 | 重量/サイズ推定、Policy候補、要確認 |
| 実送料差額の未回収 | 利益率低下 | 推定送料と実送料をSKU単位で比較し、次回Policyへ戻す |
| 関税請求のズレ | 利益率低下、請求確認漏れ | 申告価格、HTS候補、想定関税額、請求額を記録 |
| Chatwork情報の反映漏れ | 品切れ/値上げ/廃番の見落とし | 手動貼付またはAPI差分取得で候補化 |
| 規約違反/禁止品 | 出品停止 | 禁止リスト、カテゴリ別チェック |
| 価格秩序崩れ | 関係先との問題 | 基準セラー総額下回りチェック |
| 出品CSVミス | 大量誤出品 | 10件テスト、差分確認、承認後アップロード |
| AIの誤判定 | 誤出品 | AIは候補表示まで、人間確認 |
| 高頻度アクセス | サイト側負荷/ブロック | 分散実行、低頻度、手動確認併用 |

## 90日改善ロードマップ

### 0-2週間

- パートさん向け手順書を整える。
- 価格監視MVPを本番シートへ追加する。
- 送料差額、関税、Chatwork情報の追加タブを作る。
- 10件、50件、100件で価格取得テストをする。
- Sellsta CSV実出品テストを完了する。
- 要確認キュー運用を始める。

### 3-4週間

- 出品CSV生成を安定させる。
- AIタイトル/説明文生成を試す。
- Shipping Policy候補を出す。
- 送料差額と関税差額を既存出品SKUへ突合する。
- Chatworkの品切れ/値上げ情報を `出品反映候補` に入れる。
- 既存出品と価格監視マスターを突合する。
- 赤字化/取扱終了リストを作る。

### 2か月目

- Product Researchで売れ筋/薄利/低回転を分類する。
- A/B/Cランク運用を始める。
- 外注出品作業をCSV確認中心に変える。
- 価格改定候補CSV、出品停止候補CSVを出す。

### 3か月目

- Seller Hub Reports形式にも対応する。
- 出品から価格監視までを週次ダッシュボード化する。
- 低回転・薄利商品の整理を始める。
- AIによるサイズ/重量推定を改善する。

## 三神さんへの提案

最短で効く順番:

1. 価格監視と取扱終了検知で赤字事故を止める。
2. 実送料差額と関税差額を候補化し、利益率を守る。
3. Chatworkの品切れ・値上げ情報を出品リストへ戻す。
4. パートさんの入力品質を揃える。
5. Sellsta CSVの出品テストを完了する。
6. 出品外注の作業を「手入力」から「例外確認」に変える。
7. 売れ筋だけをAランク化して、出品数を増やす。
8. 既存出品の価格/Shipping Policyを一括見直しする。

今の段階で最も大事なのは、出品数を増やす前に「赤字にならない監視」と「作業者が迷わない入力ルール」を先に固めること。

## 参照した公式情報

- eBay Bulk Listing Tools: Seller Hub Reports、MIP、Trading API、Sell Feed API、Bulk Listing Toolの使い分け。
- eBay Seller Hub Reports: CSV/Excelテンプレートで追加、修正、再出品、終了が可能。
- eBay Product Research: 直近3年の販売データ、平均販売価格、送料、Sell-throughなどを確認可能。
- eBay Selling vehicles, parts, and accessories: Parts compatibility/Fitmentはカーパーツの検索性と返品リスクに関係する。
- Chatwork API/ヘルプ: メッセージ一覧は最大100件、差分取得またはWebhook運用が前提。
- USITC HTS / CBP: 関税率確認はHTSを基準にし、最終判断はCBP側で行われる。
- モノタロウ利用規約: 掲載価格、仕様、取扱は事前通知なく変更/中止される場合がある。
