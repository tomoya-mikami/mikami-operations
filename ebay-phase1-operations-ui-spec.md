# eBay業務自動化 Phase1 操作画面・UI仕様案

最終更新: 2026-05-26

## 結論

Phase1はGASエディタで関数を探して実行する運用だと分かりづらい。

今後は「スプレッドシート上のメニュー/ボタン」または「Webダッシュボード」から、処理状況確認、ドライラン、月次チェックを実行できる形へ寄せる。

## 目的

- 三神さんがGASエディタを開かなくても状態確認できる。
- どの機能が何をするか分かる。
- 誤って削除・移動・通知・上書きしない。
- 月次業務のどこで止まっているか分かる。

## 基本方針

- 既存GAS 12ファイルは壊さない。
- 最初は既存機能を直接改修せず、操作入口とログ表示を整える。
- 危険操作は必ず確認画面またはドライランを挟む。
- 削除、Drive移動、トリガー変更、Script Properties変更は画面から直接実行しない。

## 操作区分

### 画面から実行してよい

- 状態確認。
- 件数確認。
- フォルダ/シート存在確認。
- ログ表示。
- ドライラン。
- 通知なしのチェック処理。

### 実行前に確認が必要

- Driveファイル移動。
- Driveファイル名変更。
- 本番シートへの大量書き込み。
- 通知送信。
- チェックリスト復元。
- トリガー変更。
- GASファイル削除。

### 画面から直接実行しない

- Google Driveファイル削除。
- GASファイル削除。
- Script Properties変更。
- APIキー/トークン表示。
- スプレッドシート行削除。

## Webダッシュボード化する場合の注意

- まずは表示とドライラン中心にする。
- 削除・移動・通知は確認画面を挟む。
- GAS本体を大きく作り替えず、既存シートを参照する構成にする。
- 使わない画面を増やさない。
- 三神さんが見るべき数字とエラーだけを最初に出す。
